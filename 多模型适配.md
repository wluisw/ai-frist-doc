# 多模型适配指南

> AI First Demo harness v2.1 升级：模型无关——支持 Anthropic / OpenAI / DeepSeek / Qwen / Kimi / GLM 等任意厂商。
> **切厂商只需 2 个 env：`LLM_PROVIDER` + `LLM_API_KEY`，代码零改动。**

## 快速切换

```bash
# Anthropic（默认）
LLM_PROVIDER=anthropic
LLM_API_KEY=sk-ant-...

# OpenAI
LLM_PROVIDER=openai
LLM_API_KEY=sk-...

# DeepSeek
LLM_PROVIDER=deepseek
LLM_API_KEY=sk-...

# Qwen（阿里云百炼）
LLM_PROVIDER=qwen
LLM_API_KEY=sk-...

# Kimi（Moonshot AI）
LLM_PROVIDER=kimi
LLM_API_KEY=sk-...

# GLM（智谱 AI）
LLM_PROVIDER=glm
LLM_API_KEY=<your-key>
```

## 支持的厂商与端点

| provider | 基础 URL | 默认模型 |
|----------|----------|---------|
| `anthropic` | https://api.anthropic.com | claude-sonnet-4-6 |
| `openai` | https://api.openai.com/v1 | gpt-4o |
| `deepseek` | https://api.deepseek.com/v1 | deepseek-chat |
| `qwen` | https://dashscope.aliyuncs.com/compatible-mode/v1 | qwen-turbo |
| `kimi` | https://api.moonshot.cn/v1 | moonshot-v1-8k |
| `glm` | https://open.bigmodel.cn/api/paas/v4 | glm-4 |

可通过 `LLM_BASE_URL` 覆盖基础 URL（用于自建代理或私有部署）。

## 可选覆盖

```bash
LLM_BASE_URL=https://your-proxy.example.com/v1   # 自定义端点
LLM_MODEL=claude-opus-4-7                         # 覆盖默认模型
LLM_MAX_TOKENS=4096                               # 覆盖最大 token
```

## 设计原则

### 适配器模式

所有厂商相关代码隔离在 `LLMProvider` 适配器接口后。业务代码（agent scripts、
`scripts/goal_loop.py`、`scripts/ai_review.py`、`scripts/triage_engine.py`）
只调用统一接口，不感知厂商细节：

```python
# scripts/_adapters.py — 统一入口
from scripts._adapters import get_llm_client

client = get_llm_client()          # 读 LLM_PROVIDER 自动选厂商
response = await client.complete(
    model=os.getenv("LLM_MODEL"),
    messages=[{"role": "user", "content": prompt}],
)
```

### Fail-safe

- `LLM_PROVIDER` 未设置时默认 `anthropic`
- `LLM_API_KEY` 未设置时启动即报错（不静默失败）
- provider 不支持的能力（如 function calling）在适配层返回明确错误，不传到业务逻辑

### Sub-agent 配置中的模型分层

`.claude/agents/*.toml` 中的 `model` 字段仍可指定具体模型 ID，但推荐用
`LLM_MODEL` env 统一覆盖，方便批量切换到同一厂商的不同模型：

```toml
# .claude/agents/implementer.toml 示例
[agent]
name = "implementer"
model = "claude-sonnet-4-6"   # 被 LLM_MODEL env 覆盖
```

## 切换厂商检查清单

1. 设置 `LLM_PROVIDER` 和 `LLM_API_KEY`
2. 确认目标厂商的 `context window` 满足 harness 需求（推荐 ≥ 32k tokens）
3. 确认厂商支持 **流式输出**（harness goal-loop 依赖 SSE streaming）
4. 跑 `scripts/test_ai_review_passes.py` 验证 AI 评审流程正常
5. 跑 `pnpm test` 确认前端单测不受影响（前端不直接调 LLM）

## 常见问题

**Q: 前端页面怎么切换模型？**  
前端（`apps/web`）不直接调 LLM，`LLM_PROVIDER` / `LLM_API_KEY` 是后端 / harness scripts 的
环境变量。前端只通过特性开关（`VITE_FLAG_*`）控制 UI 行为。

**Q: 我想同时用两个厂商（如主力 Anthropic + 备用 OpenAI）怎么办？**  
在 `scripts/_adapters.py` 中实现 `FallbackProvider`：主厂商超时或报错时自动切备用。
当前 v2.1 不内置此功能，欢迎 PR。

**Q: Qwen / Kimi / GLM 兼容 OpenAI SDK 吗？**  
这三者均提供 OpenAI 兼容端点，设 `LLM_PROVIDER=openai` + 对应 `LLM_BASE_URL` 也可工作，
但推荐用具名 provider 以便日后统一管理和指标归因。
